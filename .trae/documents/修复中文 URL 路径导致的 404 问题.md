用户的需求是优化后台文章发布体验，特别是针对直接粘贴 Markdown 的场景。

我们将对 `PostForm` 和后台逻辑进行如下重大升级：

1.  **非必填 Slug (URL)**:
    *   前端：`Slug` 输入框不再设为 `required`。
    *   后端：`createPost` 接收到空 Slug 时，自动生成一个随机短 ID（例如 `post-xyz123`），避免因空值或中文导致错误。

2.  **智能提取标题与描述**:
    *   前端：当您粘贴 Markdown 到正文框时，如果标题为空，系统会自动提取正文的第一行作为标题。
    *   前端：同时自动提取正文前 150 个字符作为描述。
    *   效果：您只需粘贴正文，其他字段自动填充，极大提高效率。

3.  **交互式标签管理**:
    *   前端：引入新的 `TagInput` 组件，替代现在的纯文本框。
    *   功能：输入标签后按回车生成气泡，支持点击 `x` 删除，体验更现代、直观。

4.  **取消不必要的必填项**:
    *   除了标题和正文，其他字段尽量放宽限制，由系统自动兜底。

这将使您的写作流程变成：**打开后台 -> 粘贴文章 -> 点击发布**，一气呵成。
