您希望对文章详情页进行全面的视觉和功能升级，具体需求包括：
1.  **生成目录结构**：在左侧（或右侧）空白处生成文章目录。
2.  **极简 UI 视觉**：优化字号（不要太大），调整分割线。
3.  **还原换行风格**：保留原文的换行，实现“句号即换行”的排版效果。

### 实施计划

#### 1. 目录功能 (TOC) 实现
*   **依赖安装**：安装 `rehype-slug` (为标题生成 ID) 和 `github-slugger` (提取标题数据)。
*   **工具函数**：编写 `src/lib/toc.ts`，用于从 Markdown 文本中提取 H2/H3 标题。
*   **组件开发**：创建 `src/components/TableOfContents.tsx`，实现一个悬浮（Sticky）的目录导航。
*   **MDX 配置**：在 `BlogPage` 中配置 `rehypePlugins`，确保渲染出的标题带有 ID，以便跳转。

#### 2. 视觉排版优化
*   **布局重构**：将文章页布局改为 **Grid** 或 **Flex** 布局。
    *   **左侧/右侧**：放置目录（TOC）。
    *   **中间**：文章正文。
*   **排版微调 (`prose` 配置)**：
    *   **字号**：调整 `prose-lg` 为更精致的 `prose-base` 或自定义字号，避免过大。
    *   **分割线**：在标题和正文之间添加精致的短分割线。
    *   **颜色**：使用更柔和的 slate 灰色系，减少视觉疲劳。

#### 3. 换行风格还原
*   Markdown 默认会合并相邻的换行符。为了“最大化还原原文”，我们需要配置 MDX 渲染器，使其**保留换行** (Hard Breaks)。
*   **方案**：引入 `remark-breaks` 插件。它会将 Markdown 中的每一个换行符都转换为 `<br>`，从而实现“句号后回车即换行”的效果。

执行完毕后，您的文章页将拥有清晰的导航目录、更舒适的阅读字号，以及完全忠实于原文的段落排版。
