Based on your request to optimize code structure and logic for better maintainability and error prevention, I propose the following refactoring plan. This focuses on removing "magic strings" (hardcoded values like 'blog', 'ai', 'video') and reducing code duplication.

### 1. Centralize Constants & Types

Create a new file `src/lib/constants.ts` to define system-wide constants. This ensures that if we ever change a category name or add a type, we only need to update it in one place.

* Define `PostType` (blog, resource)

* Define `PostCategory` (ai, content-creation)

* Define `ResourceType` (article, video, podcast, tool)

### 2. Refactor Server Actions (`src/app/actions/content.ts`)

The current `createPost` and `updatePost` functions have duplicated logic for extracting and validating form data.

* Create a helper function `parsePostFormData` to handle data extraction and default value logic (like slug generation).

* Replace hardcoded strings with the new constants.

* Improve type safety for form inputs.

### 3. Update Admin Components (`src/components/admin/PostForm.tsx`)

Update the form component to use the constants for dropdown menus and logic checks.

* Replace `'blog'`, `'resource'`, etc., with imported constants.

* This prevents typos (e.g., typing 'video' vs 'videos') from causing bugs.

### 4. Update Frontend Logic (`src/components/ResourceFilter.tsx`)

* Update the filtering logic to use the same constants, ensuring the frontend filter always matches the backend data structure.

### Why this helps:

* **Reduces Errors**: No more risk of typos in string values (e.g., 'content-creation' vs 'content\_creation').

* **Easier Maintenance**: Adding a new category or resource type becomes much easier.

* **Cleaner Code**: Removes duplication in server actions.
