您希望对文章详情页的布局进行精细调整，以解决“右边太空”、“正文不够居中”以及“整体不够精致”的问题。

### 核心问题分析
1.  **右侧太空**：目前的布局是 `左(TOC) - 中(文章) - 右(空)`，由于右侧只是一个占位 `div`，导致在宽屏下视觉重心偏左，右侧显得空荡。
2.  **正文位置**：您希望正文“靠右一点”，实际上是希望正文在整个页面中更加居中或平衡，不被左侧目录挤得太偏。
3.  **视觉精致度**：标题字号还需要更小，目录整体需要更纤细（小字号）。

### 优化方案 (Grid 布局重构)
为了实现更完美的平衡，我将把原本的 **Flex 布局** 改为 **CSS Grid 布局**。Grid 能更精确地控制列宽比例。

1.  **整体容器**：使用 12 列 Grid (`grid-cols-12`)。
2.  **左侧目录**：占据 2 列 (`col-span-2`)。
    *   **样式微调**：字号调小 (`text-xs` / `text-sm`)，行间距调小，让目录看起来更精致。
3.  **中间正文**：占据 8 列 (`col-span-8`)，并限制最大宽度居中。
    *   **位置调整**：将正文容器在网格内居中，或者稍微向右偏移（通过 `mx-auto` 或 `ml-auto`）。
    *   **标题优化**：进一步缩小 `h1` 字号。
4.  **右侧空间**：占据 2 列 (`col-span-2`)。
    *   **利用空间**：虽然现在留空，但 Grid 布局会自动平衡左右边距，不会显得“空”，而是显得“留白”。

### 具体实施步骤
1.  **修改 `src/app/blog/[slug]/page.tsx`**：
    *   将 Flex 容器改为 Grid 容器。
    *   调整 `h1` 类名，减小字号。
    *   调整左右列宽比例。
2.  **修改 `src/components/TableOfContents.tsx`**：
    *   调整文字大小 (`text-xs` 或更小的 `text-sm`)。
    *   调整内边距，使其更紧凑。

### 预期效果
页面将呈现经典的“三栏平衡”布局：
`[ 目录 (紧凑) ]  [     正文 (居中, 视觉焦点)     ]  [ 留白 (平衡) ]`
正文不再被挤在左边，目录更加精致不抢眼。
