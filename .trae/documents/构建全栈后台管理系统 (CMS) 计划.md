为了实现一个可编辑所有内容（包括文章、资源、以及页面上的固定文案）的后台管理系统，我们需要将目前的“静态文件存储（MDX）+ 硬编码文案”架构，升级为“数据库驱动 + 动态渲染”架构。

以下是详细的实施计划：

## 1. 核心架构升级
我们将引入数据库和后台管理逻辑，使内容与代码分离。
- **数据库 (Database)**: 使用 **Prisma (ORM)** + **SQLite** (开发环境) / **Postgres** (生产环境)。
- **身份验证 (Auth)**: 使用 **NextAuth.js** 保护后台管理路由 (`/admin`)。
- **后台 UI**: 使用 **Tailwind CSS** + **React Hook Form** 构建管理界面。

## 2. 数据库模型设计 (Schema Design)
我们需要设计以下数据模型来覆盖现有内容：
- **Post (文章/资源)**: 替代现有的 MDX 文件。包含 `title`, `slug`, `content`, `category`, `tags`, `type` (blog/resource) 等字段。
- **GlobalConfig (全站配置)**: 存储页眉、页脚、SEO 信息等全局内容。
- **PageContent (页面文案)**: 存储首页 Hero 标题、关于页介绍等目前硬编码的文本。设计为 Key-Value 结构，例如 `page: 'home', section: 'hero', key: 'title', value: '...'`。

## 3. 实施步骤

### 第一阶段：基础设施搭建
1.  **安装依赖**: 配置 Prisma, NextAuth.js。
2.  **定义 Schema**: 编写 `schema.prisma` 文件，定义上述数据模型。
3.  **数据迁移脚本**: 编写脚本读取现有的 `content/` 目录下的 MDX 文件，将其解析并批量导入到数据库中，确保数据不丢失。

### 第二阶段：后台管理系统开发 (`/admin`)
1.  **登录页**: 创建管理员登录界面。
2.  **内容管理页**:
    -   **文章/资源列表**: 展示所有内容，支持增删改查。
    -   **编辑器**: 集成 Markdown 编辑器（如 `react-markdown-editor-lite` 或类似），支持实时预览。
3.  **页面文案管理**: 一个表单页面，列出所有可配置的页面文案（如首页标题、关于页简介），允许直接修改文本。

### 第三阶段：前端重构 (Frontend Refactor)
1.  **替换数据源**: 修改 `src/lib/mdx.ts` 或新建 `src/lib/db.ts`，将原来的文件读取逻辑改为数据库查询逻辑。
2.  **动态化组件**: 修改 `Home`, `About`, `Footer` 等组件，不再直接写死文字，而是从数据库获取配置信息渲染。

## 4. 预期成果
完成后的系统将具备：
-   **`/admin` 入口**: 只有管理员可访问。
-   **全内容编辑**: 既能写博客，也能改网站标题、Slogan、联系方式等。
-   **零代码更新**: 修改文案或发布新资源无需修改代码或重新部署（如果使用独立数据库）。
